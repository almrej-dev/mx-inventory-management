    1. Create `src/schemas/stock.ts` -- Zod schema for stock receiving:

       - receivedDate: z.date() or z.string().pipe(z.coerce.date()) for date input

       - notes: z.string().max(500).optional()

    2. Create `src/actions/stock.ts` -- Server actions:
       - `receiveStock(rawData: unknown)`:
         - Call requireRole("staff")
         - Validate with receivingSchema.safeParse(rawData)
         - Load the item to get cartonSize and unitWeightMg: prisma.item.findUnique({ where: { id: data.itemId } })
         - If item not found or soft-deleted, return { error: "Item not found" }
         - Calculate quantity in storage units:
           - For PACKAGING type: quantityUnits = data.quantityCartons * item.cartonSize (pieces)
           - For all other types: quantityMg = data.quantityCartons * item.cartonSize * item.unitWeightMg (milligrams)
         - Calculate total cost: totalCostCentavos = pesosToCentavos(data.costPesos) * data.quantityCartons
         - Use prisma.$transaction([...]) to ATOMICALLY:
           1. prisma.inventoryTransaction.create({ data: { itemId, type: "RECEIVE", quantity: calculatedQty, costCentavos: totalCostCentavos, notes, createdBy: user.id, referenceId: generate a unique ref like `RCV-${Date.now()}` } })
           2. prisma.item.update({ where: { id: itemId }, data: { stockQty: { increment: calculatedQty } } })
         - Also update the item's costCentavos if the new cost differs (latest purchase cost)
         - Return { success: true } or { error: string }
         - revalidatePath("/stock/receiving")
         - revalidatePath("/items")

       - `getTransactionHistory(filters?: { itemId?: number, type?: string, limit?: number })`:
         - Call requireRole("viewer")
         - Query prisma.inventoryTransaction.findMany with filters, ordered by createdAt desc
         - Include item relation for display (item name, SKU)
         - Default limit: 50
         - Return transactions array

       CRITICAL: The $transaction call MUST wrap BOTH the ledger insert AND the stock_qty increment. Never do one without the other. This is the foundational pattern that all stock mutations in future phases will follow.

         - Shows item name + SKU + type
         - Only show active items (not soft-deleted)
       - Quantity: number input labeled "Cartons"
       - When item is selected AND quantity is entered, show real-time conversion:
         - For weight items: "{qty} cartons = {qty * cartonSize} units = {qty * cartonSize * unitWeightGrams}g"
         - For packaging: "{qty} cartons = {qty * cartonSize} pieces"
       - Received Date: date input (default to today)
       - Cost per Carton: number input labeled "PHP per carton"
         - Show total: "Total: PHP {costPerCarton * qty}"
       - Notes: textarea (optional)
       - Submit button: "Record Receiving"
       - On success: show success message, reset form
       - On error: show error message
       - Use shadcn/ui components: Select/Combobox for item, Input for numbers, Calendar/DatePicker or simple Input type="date" for date, Textarea for notes
