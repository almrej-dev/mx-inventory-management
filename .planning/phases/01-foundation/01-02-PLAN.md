  <action>
    1. Create `src/schemas/item.ts` -- Zod schema for item creation and editing:
       - name: z.string().min(1, "Name is required").max(100)
       - sku: z.string().min(1, "SKU is required").max(50)
       - type: z.enum(["RAW_MATERIAL", "SEMI_FINISHED", "FINISHED", "PACKAGING"])
       - category: z.string().max(50).optional().or(z.literal(""))
       - unitWeightGrams: z.number().positive("Weight must be positive") -- NOTE: form collects GRAMS, action converts to mg
       - cartonSize: z.number().int().positive("Carton size must be at least 1")
       - costPesos: z.number().nonnegative("Cost cannot be negative") -- NOTE: form collects PESOS, action converts to centavos
       - minStockQty: z.number().int().nonnegative().default(0)
       - Export type `ItemFormData = z.infer<typeof itemSchema>`
       - IMPORTANT: The form schema uses user-friendly units (grams, pesos). The server action converts to storage units (mg, centavos) before database write.

    2. Create `src/actions/items.ts` -- Server actions:
       - `createItem(rawData: unknown)`:
         - Call requireRole("staff") -- staff and admin can create
         - Validate with itemSchema.safeParse(rawData)
         - Convert: unitWeightMg = gramsToMg(data.unitWeightGrams), costCentavos = pesosToCentavos(data.costPesos)
         - prisma.item.create({ data: { ...converted values } })
         - Return { success: true, item } or { error: fieldErrors }
         - revalidatePath("/items")

       - `updateItem(id: number, rawData: unknown)`:
         - Call requireRole("staff")
         - Same validation and conversion as createItem
         - prisma.item.update({ where: { id }, data: { ...converted } })
         - revalidatePath("/items")

       - `deleteItem(id: number)`:
         - Call requireRole("staff")
         - Soft delete: prisma.item.update({ where: { id }, data: { deletedAt: new Date() } })
         - revalidatePath("/items")

       - `getItems(filters?: { search?: string, type?: string, category?: string })`:
         - Call requireRole("viewer") -- all roles can read
         - Build Prisma where clause: { deletedAt: null, AND: [...filter conditions] }
         - If search: name contains (case-insensitive) OR sku contains
         - If type: filter by type enum value
         - If category: filter by category
         - Return items array ordered by name
         - Convert display values in the response: include weightGrams and costPesos computed fields

       - `getItem(id: number)`:
         - Call requireRole("viewer")
         - prisma.item.findUnique({ where: { id } })
         - Return null if not found or soft-deleted

    3. Create `src/components/items/item-form.tsx` -- Shared form for create and edit:
       - Client component ("use client")
       - Props: initialData (optional, for edit mode), onSubmit callback
       - Uses React Hook Form with zodResolver(itemSchema)
       - Fields rendered with shadcn/ui components:
         - Name (Input)
         - SKU (Input) with link to SKU legend page
         - Type (Select dropdown with four options from constants.ts)
         - Category (Input, optional)
         - Unit Weight (Input type="number", label says "grams") -- user enters grams, stored as-is in form state
         - Carton Size (Input type="number", label says "units per carton")
         - Cost (Input type="number", label says "PHP") -- user enters pesos, stored as-is in form state
         - Minimum Stock Quantity (Input type="number")
       - Show a computed summary: "1 carton = {cartonSize} units = {cartonSize * unitWeightGrams}g"
       - When in edit mode, pre-populate fields converting from storage units: unitWeightGrams = mgToGrams(item.unitWeightMg), costPesos = centavosToPesos(item.costCentavos)
       - Submit button text: "Create Item" or "Update Item" based on mode
       - Show form errors inline under each field

    4. Create `src/app/(dashboard)/items/new/page.tsx`:
       - Server component wrapper
       - Renders ItemForm with onSubmit wired to createItem action
       - On success, redirect to /items with success toast/message

    5. Create `src/app/(dashboard)/items/[id]/edit/page.tsx`:
       - Server component that loads item by ID via getItem()
       - If item not found, show 404 or redirect
       - Renders ItemForm with initialData pre-populated
       - On success, redirect to /items

    IMPORTANT: Form collects user-friendly units (grams, pesos). Server action converts to storage units (milligrams, centavos). Display helpers convert back for the UI. The Prisma schema stores ONLY integers.

    IMPORTANT: Every mutation action calls requireRole("staff") -- not just middleware route protection.
