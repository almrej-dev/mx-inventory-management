<task type="auto">
  <name>Task 2: Create BOM explosion library and recipe Zod schema</name>
  <files>src/lib/bom.ts, src/schemas/recipe.ts</files>
  <action>
1. Create `src/lib/bom.ts` with two main functions:

   **`explodeBom(parentItemId: number, visited?: Set<number>): Promise<BomLine[]>`**
   - Fetches `recipeIngredient.findMany({ where: { parentItemId }, include: { childItem: { select: { id, name, sku, type, costCentavos, unitWeightMg } } } })`
   - For each ingredient:
     - If `childItem.type === 'SEMI_FINISHED'`, check if it has its own recipe (`recipeIngredient.count({ where: { parentItemId: childItem.id } })`)
     - If yes, recurse into `explodeBom(childItem.id, new Set(visited))` and add returned lines (these are already leaf-level)
     - If no (leaf node), create a BomLine entry
     - For RAW_MATERIAL and PACKAGING, create BomLine directly
   - **Circular reference guard:** Before recursing, check if `parentItemId` is in `visited` set. If yes, throw `Error('Circular recipe detected for item ${parentItemId}')`. Add current ID to visited before recursing.
   - After collecting all lines, **merge duplicates**: group by `itemId`, sum `quantityMg` and `quantityPieces`
   - Compute `lineCostCentavos` for each merged line:
     - For weight-based items (RAW_MATERIAL, SEMI_FINISHED without recipe): `Math.round(quantityMg * unitCostCentavos / unitWeightMg)` -- cost proportional to weight used vs unit weight
     - For PACKAGING: `quantityPieces * unitCostCentavos` -- cost per piece (costCentavos is per unit for packaging)
   - Return sorted by itemName

   **`calculateRecipeCost(parentItemId: number): Promise<number>`**
   - Calls `explodeBom(parentItemId)`
   - Returns sum of all `lineCostCentavos` values

   **`checkCircularReference(parentItemId: number, newChildItemId: number): Promise<boolean>`**
   - Checks if adding `newChildItemId` as an ingredient of `parentItemId` would create a cycle
   - If `newChildItemId === parentItemId`, return true (direct self-reference)
   - If `newChildItemId` is SEMI_FINISHED, check if `parentItemId` appears anywhere in `newChildItemId`'s recipe tree (recursive walk)
   - Returns true if circular, false if safe

   **`BomLine` interface:**
   ```typescript
   export interface BomLine {
     itemId: number;
     itemName: string;
     itemSku: string;
     itemType: string;
     quantityMg: number;
     quantityPieces: number;
     unitCostCentavos: number;
     unitWeightMg: number;
     lineCostCentavos: number;
   }
   ```

   Import `prisma` from `@/lib/prisma`. Use integer-only arithmetic throughout -- no floating point.

2. Create `src/schemas/recipe.ts` with Zod v4 validation:
   - Import from `zod/v4` (NOT `zod`)
   - `ingredientLineSchema`: object with `childItemId` (number, int, positive), `quantityGrams` (number, nonnegative -- for RAW_MATERIAL/SEMI_FINISHED, converted to mg in action), `quantityPieces` (number, int, nonnegative -- for PACKAGING)
   - `recipeSchema`: object with `parentItemId` (number, int, positive), `ingredients` (array of ingredientLineSchema, min 1 "At least one ingredient required")
   - Export `RecipeFormData = z.infer<typeof recipeSchema>`
   - Note: The form schema uses `quantityGrams` (user-friendly) which the server action will convert to `quantityMg` using `gramsToMg()`, consistent with the Phase 1 item form pattern.
