<task type="auto">
  <name>Task 2: Create processSalesLines server action with aggregated BOM deduction and history queries</name>
  <files>

  <action>
Create src/actions/sales.ts following the pattern from src/actions/stock.ts (use "use server" directive, requireRole guard, try/catch with error returns, revalidatePath).

1. processSalesLines(rawData: unknown):
   - requireRole("staff")
   - Parse with processSalesSchema.safeParse(rawData), return { error } on failure
   - PHASE 1 (outside transaction): BOM explosion and aggregation
     - For each line, call explodeBom(line.itemId) from src/lib/bom.ts
     - Build deductionMap: Map&lt;number, { itemId, totalQtyMg, totalQtyPieces, itemType }&gt;
     - For each BomLine, multiply quantities by line.quantity (critical: BOM returns per-unit, must scale by units sold)
     - To get itemType for the deduction, load items in bulk with prisma.item.findMany({ where: { id: { in: [...deductionMap.keys()] } }, select: { id, type } })
   - PHASE 2 (single prisma.$transaction): All writes atomically
     - Create SalesUpload record (fileName, source, saleDate as new Date(), totalLines, matchedLines = lines.length, status "completed", createdBy user.id)
     - Create SalesLine records with createMany (uploadId, itemId, productName, quantity, unitPriceCentavos, deducted: true)
     - For each entry in deductionMap:
       - Determine deductQty: PACKAGING uses totalQtyPieces, all others use totalQtyMg
       - Create InventoryTransaction with type SALE_DEDUCTION, quantity: -deductQty, referenceId: "SALE-{uploadId}", notes: "Auto-deducted from sales batch #{uploadId}", createdBy: user.id
       - Update item stockQty with decrement: deductQty
   - revalidatePath for "/sales", "/stock/history", "/items"
   - Return { success: true, uploadId: result.id }
   - Wrap in try/catch, return { error: message } on failure

2. getSalesUploads():
   - requireRole("viewer")
   - Query salesUpload.findMany ordered by createdAt desc, include _count of lines
   - Return { uploads }

3. getSalesUploadDetail(uploadId: number):
   - requireRole("viewer")
   - Query salesUpload.findUnique with include lines (include item { name, sku, type })
   - Return { upload } or { error: "Not found" }

4. getFinishedItems():
   - requireRole("staff")
   - Query items where type is FINISHED, deletedAt is null, AND the item has at least one recipeIngredient (i.e., has a recipe defined)
   - Return { items } with id, name, sku fields
   - This is used by the upload preview and manual entry form to show only sellable products

CRITICAL per research: explodeBom() calls happen OUTSIDE the transaction to avoid holding the transaction open during recursive DB reads. Only writes go inside the transaction.

CRITICAL per research: Allow negative stock -- do NOT check if stockQty would go below 0. The system must record reality.
