   - Include a date input for saleDate (default to today) since POS exports may not have dates per row
   - Show file name and row count after parsing

   - Match each row's productIdentifier against finished items by:
     - Exact SKU match (case-insensitive)
     - Exact name match (case-insensitive)

   - Display a summary bar at top: "X rows matched, Y rows with errors, Z total"
   - Show rows in a simple HTML table (not TanStack Table -- this is a one-time preview, not a reusable data table):
     - Columns: Row #, Product (from CSV), Matched Item (from system or "Not found" in red), Quantity, Errors
     - Error rows highlighted with red-50 background
     - Valid rows have green checkmark icon
   - "Confirm and Process" button (disabled if zero valid rows):
     - On click, build the processSalesLines input from valid rows only:

     - Use useTransition for pending state (per project pattern from 02-02)
