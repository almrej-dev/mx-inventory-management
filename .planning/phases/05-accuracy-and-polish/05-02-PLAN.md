1. In `src/schemas/stock.ts`, add reconciliation schemas below the existing waste schema:
   ```
   export const reconciliationCountSchema = z.object({
     itemId: z.number().int().positive(),
     physicalCount: z.number().nonnegative("Count cannot be negative"),
   });

   export const reconciliationSchema = z.object({
     counts: z.array(reconciliationCountSchema).min(1, "Enter at least one count"),
     notes: z.string().max(500).optional(),
   });

   export type ReconciliationFormData = z.infer<typeof reconciliationSchema>;
   ```

2. In `src/actions/stock.ts`, add two new server actions:

   **getItemsForReconciliation():**
   - Call `requireRole("staff")`
   - Load all active items: `prisma.item.findMany({ where: { deletedAt: null }, select: { id, name, sku, type, stockQty }, orderBy: { name: "asc" } })`
   - Return `{ items }` -- the client form will convert stockQty to display units
   - Wrap in try/catch returning `{ error, items: [] }` on failure

   **submitReconciliation(rawData: unknown):**
   - Call `requireRole("staff")` to get user
   - Parse with `reconciliationSchema.safeParse(rawData)`
   - Generate batch reference: `const batchRefId = "RECON-" + Date.now()`
   - Load current items with `prisma.item.findMany({ where: { id: { in: itemIds } }, select: { id, type, stockQty } })`
   - Build a Map for quick lookup: `new Map(items.map(i => [i.id, i]))`
   - For each count entry:
     a. Find item in map, skip if not found
     b. Convert physicalCount to storage units: PACKAGING stays as pieces; all others use `gramsToMg(count.physicalCount)` since user enters grams
     c. Calculate variance: `physicalStorageQty - item.stockQty`
     d. Skip if variance is 0 (no discrepancy)
     e. For discrepant items, push two operations to ops array:
        - `prisma.inventoryTransaction.create({ data: { itemId, type: "ADJUSTMENT", quantity: variance, referenceId: batchRefId, notes: data.notes ? "Reconciliation: " + data.notes : "Physical count reconciliation", createdBy: user.id } })`
        - `prisma.item.update({ where: { id: itemId }, data: { stockQty: physicalStorageQty } })` -- SET to physical count, NOT increment/decrement
   - If no ops (all items match): return `{ success: true, message: "No discrepancies found." }`
   - Execute `await prisma.$transaction(ops)`
   - Call `revalidatePath` for: "/stock/reconciliation", "/stock/history", "/items", "/"
   - Return `{ success: true, message: "Reconciliation complete. X items adjusted." }` where X is the count of adjusted items (ops.length / 2)
   - Wrap in try/catch

CRITICAL: Reconciliation uses absolute SET for stockQty (`stockQty: physicalStorageQty`), NOT increment/decrement. The physical count IS the truth. The ledger entry records the variance (delta) for audit trail.

CRITICAL: physicalCount from form is in user-friendly units (grams for weight, pieces for packaging). Must convert to storage units (mg for weight) before comparing with stockQty.
