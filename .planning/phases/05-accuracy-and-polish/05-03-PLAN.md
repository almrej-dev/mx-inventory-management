After the `findMany`, collect all unique `createdBy` UUIDs from the transactions, then batch-query the `Profile` model to resolve display names:

```typescript
// Collect unique user IDs
const userIds = [...new Set(transactions.map(tx => tx.createdBy))];

// Batch-query profiles for display names
const profiles = await prisma.profile.findMany({
  where: { id: { in: userIds } },
  select: { id: true, fullName: true },
});

const profileMap = new Map(profiles.map(p => [p.id, p.fullName]));

// Attach user display name to each transaction
const enriched = transactions.map(tx => ({
  ...tx,
  userName: profileMap.get(tx.createdBy) ?? tx.createdBy.slice(0, 8),
}));

return { transactions: enriched };
```

The fallback `tx.createdBy.slice(0, 8)` shows a truncated UUID if the profile is somehow missing, preventing a blank cell.

Do NOT add a Prisma relation between InventoryTransaction and Profile -- the `profiles` table is managed by Supabase auth triggers and adding a Prisma relation would require a migration. A separate lookup query is the correct pattern here.
