Implement two exported async functions:

1. `getReportFilterOptions()` -- requireRole("viewer"). Returns:
   - products: prisma.item.findMany where type = "FINISHED" and deletedAt is null, select id and name, orderBy name asc
   - categories: prisma.item.findMany where deletedAt is null, select category, distinct. Filter out null categories.
   Return { products, categories }.

2. `getSalesReport(filters: { from?: string, to?: string, productId?: number, category?: string })` -- requireRole("viewer"). Build a Prisma where clause for salesLine.findMany:
   - If from/to provided: filter on `upload: { saleDate: { gte: startOfDay(new Date(from)), lte: endOfDay(new Date(to)) } }`. Use endOfDay() from date-fns to avoid the off-by-one pitfall documented in research.
   - If productId provided: filter on `itemId: productId`
   - If category provided: filter on `item: { category: category }`
   - Include: item (select name, sku, type, category) and upload (select saleDate, source)
   - OrderBy: upload.saleDate desc
   Return { lines } where lines is the query result.

Define and export a TypeScript type `SalesReportLine` for the return shape so the client component can use it.
