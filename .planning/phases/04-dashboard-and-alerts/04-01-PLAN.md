  <name>Task 1: Create dashboard server actions with inventory aggregation queries</name>
  <files>src/actions/dashboard.ts</files>
  <action>
Create `src/actions/dashboard.ts` with "use server" directive. Import requireRole from @/lib/auth and prisma from @/lib/prisma.

Implement four exported async functions:

1. `getDashboardSummary()` -- requireRole("viewer"). Fetch all non-deleted items with select: id, stockQty, costCentavos, minStockQty, type, unitWeightMg, cartonSize. Loop to compute:
   - totalItems: count of items
   - totalValueCentavos: sum of per-item value. For PACKAGING type: `(stockQty / cartonSize) * costCentavos`. For all others: `(stockQty / (cartonSize * unitWeightMg)) * costCentavos`. Use Math.round. Guard against division by zero (cartonWeightMg === 0).
   - lowStockCount: count where `minStockQty > 0 AND stockQty < minStockQty`
   - transactionCount: prisma.inventoryTransaction.count where createdAt >= first day of current month (compute with `new Date()` + setDate(1) + setHours(0,0,0,0))

2. `getBestsellers(limit = 5)` -- requireRole("viewer"). Use prisma.salesLine.groupBy by itemId with _sum on quantity, orderBy _sum quantity desc, take limit. Then fetch item details (id, name, sku, type) for the returned itemIds using prisma.item.findMany with where id in [...]. Build a Map from items, return array of { item, totalSold }.

3. `getLowStockItems()` -- requireRole("viewer"). Fetch items where deletedAt is null and minStockQty > 0, select id, name, sku, type, stockQty, minStockQty, orderBy stockQty asc. Filter in application code: `items.filter(i => i.stockQty < i.minStockQty)`. This handles Prisma's inability to compare two columns.

4. `getReorderRecommendations()` -- requireRole("viewer"). Fetch items where deletedAt is null and minStockQty > 0, select id, name, sku, type, stockQty, minStockQty, unitWeightMg, cartonSize, orderBy stockQty asc. Partition into two arrays:
   - reorder: items where `stockQty <= 0 OR stockQty < minStockQty`
   - surplus: items where `stockQty > minStockQty * 3`
   Return { reorder, surplus }.

IMPORTANT: minStockQty is in the same storage units as stockQty (milligrams for weight-based, pieces for packaging) per research recommendation. No unit conversion needed for comparison.

Do NOT use `$queryRaw` -- application-level computation is sufficient for this dataset size (single ice cream shop, 50-200 items).
